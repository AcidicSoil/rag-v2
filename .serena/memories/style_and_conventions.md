# Style and Conventions

- Language: TypeScript with `strict: true` in `tsconfig.json`
- Module target: CommonJS, ES2021 target
- Output config: `rootDir: src`, `outDir: dist`, declaration + source maps enabled
- Formatting style observed in source:
  - double quotes for strings/imports
  - semicolons used consistently
  - 2-space indentation
  - named exports for main functions/constants
- Naming conventions observed:
  - camelCase for variables/functions
  - PascalCase for imported SDK types
  - ALL_CAPS for constants like `AUTO_DETECT_MODEL_ID`
- Implementation patterns:
  - plugin entrypoint in `src/index.ts`
  - config schema builder isolated in `src/config.ts`
  - core logic concentrated in a single prompt preprocessor module
  - async/await used throughout for SDK calls
  - status updates surfaced through LM Studio controller objects
- Operational conventions:
  - plugin config is accessed through `ctl.getPluginConfig(configSchematics)`
  - non-image attached files are the target for document parsing/retrieval
  - debug logging uses `ctl.debug(...)`
- Notes for future edits:
  - preserve existing user-facing config field names unless intentionally migrating plugin settings
  - prefer minimal, targeted changes because plugin behavior is concentrated in a small number of files
